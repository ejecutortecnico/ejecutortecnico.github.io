Misión 2 · Lección 13

Seguridad en Bases de Datos

Objetivos

  • Comprender los principales riesgos de seguridad en bases de datos.
  • Aplicar buenas prácticas de protección de datos.
  • Configurar roles, permisos y cifrado.

Riesgos Comunes

  • Inyección SQL.
  • Accesos no autorizados.
  • Exposición de datos sensibles.
  • Configuración insegura del servidor.

Prevención de SQL Injection

-- Inseguro
"SELECT * FROM usuarios WHERE nombre = '" + nombre + "'"

-- Seguro con parámetros
cursor.execute("SELECT * FROM usuarios WHERE nombre = %s", (nombre,))

Gestión de Usuarios

  • Asignar privilegios mínimos necesarios (principio de menor privilegio).
  • Crear usuarios distintos para administración y aplicación.
  • Revocar permisos no usados.

Roles y Permisos

-- Crear rol
CREATE ROLE lector;

-- Asignar permisos
GRANT SELECT ON base_datos.* TO lector;

-- Asignar rol a usuario
GRANT lector TO usuario_app;

Cifrado

  • En tránsito: usar SSL/TLS para conexiones cliente-servidor.
  • En reposo: cifrar discos y respaldos.
  • Datos sensibles: cifrar columnas como contraseñas o documentos.

Auditoría y Logs

  • Registrar accesos de usuarios.
  • Monitorear consultas sospechosas.
  • Configurar alertas de seguridad.

Buenas Prácticas

  • Actualizar constantemente el motor de base de datos.
  • Usar contraseñas seguras y autenticación multifactor.
  • Restringir acceso a la base de datos por IP o VPN.
  • Respaldar datos regularmente.

Ejercicio Guiado

  1. Crea un usuario con permisos limitados solo de lectura.
  2. Configura un rol de editor que permita SELECT y UPDATE.
  3. Implementa consultas parametrizadas en un lenguaje de programación (ejemplo: Python con SQLite o MySQL).
