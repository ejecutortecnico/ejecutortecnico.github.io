¿Qué son las Sesiones en Software?

Una sesión es un mecanismo que permite almacenar información temporal sobre un usuario mientras interactúa con una aplicación.

  • Se inicia cuando un usuario se autentica o empieza a usar el sistema.
  • Permite mantener datos durante varias peticiones HTTP.
  • Finaliza cuando el usuario cierra sesión o expira el tiempo configurado.

Ejemplo de Uso de Sesiones

Cuando un usuario inicia sesión en una aplicación:

  • El servidor crea una sesión y guarda información (ej. id_usuario).
  • Se genera un identificador único de sesión.
  • Ese identificador se guarda en una cookie en el navegador del usuario.
  • En cada nueva petición, la aplicación reconoce al usuario gracias a esa cookie.

Flask y las Sesiones

Flask incluye soporte para sesiones mediante el objeto session:

  • Se comporta como un diccionario de Python.
  • Los datos se almacenan en el lado del cliente usando cookies firmadas.
  • Se necesita configurar una clave secreta (app.secret_key) para mayor seguridad.

Flask-Login y Sesiones

Flask-Login usa las sesiones para administrar la autenticación de usuarios:

  • Cuando el usuario inicia sesión, Flask-Login guarda su user_id en la sesión.
  • En cada petición, recupera automáticamente al usuario con current_user.
  • Permite saber si un usuario está autenticado con current_user.is_authenticated.
  • Facilita proteger rutas con el decorador @login_required.

Flujo de Sesiones con Flask-Login

  1. El usuario envía sus credenciales.
  2. Flask verifica y crea la sesión.
  3. Se guarda el identificador del usuario en la cookie de sesión.
  4. Cada petición siguiente valida la sesión automáticamente.
  5. Al cerrar sesión, la sesión se elimina.
