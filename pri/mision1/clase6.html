Modelado Entidad–Relación (E-R)

Objetivos

  • Comprender el modelo entidad–relación.
  • Diferenciar entidades, atributos y relaciones.
  • Representar gráficamente un sistema en un diagrama E-R.

¿Qué es un Modelo E-R?

Un diagrama que describe la estructura lógica de una base de datos:

  • Entidades: objetos o conceptos (Cliente, Producto).
  • Atributos: propiedades de las entidades (nombre, precio).
  • Relaciones: cómo interactúan las entidades (Cliente hace Pedido).

Entidades y Atributos

  • Entidad fuerte: tiene clave primaria propia.
  • Entidad débil: depende de otra entidad.
  • Atributos: simples, compuestos, multivaluados o derivados.

Ejemplo: Cliente (id_cliente, nombre, correo, dirección).

Relaciones

  • 1 a 1: Una persona → un pasaporte.
  • 1 a N: Un cliente → muchos pedidos.
  • N a M: Muchos estudiantes ↔ muchos cursos.

Cardinalidad

Define cuántas instancias de una entidad pueden asociarse con otra:

  • 0..1
  • 1..1
  • 0..N
  • 1..N

Ejemplo de Diagrama E-R

Sistema de ventas:

  • Cliente (id, nombre, correo)
  • Pedido (id, fecha, total)
  • Producto (id, nombre, precio)

Relaciones:

  • Cliente — hace → Pedido
  • Pedido — contiene → Producto

Entidad Débil

Ejemplo: DetallePedido depende de Pedido y Producto.

Pedido(id, fecha, cliente_id)
Producto(id, nombre, precio)
DetallePedido(pedido_id, producto_id, cantidad)

De E-R a Tablas

Transformación:

  • Entidad → Tabla.
  • Atributo → Columna.
  • Relación → Clave foránea o tabla intermedia.

Herramientas para Diagramas E-R

  • MySQL Workbench
  • draw.io
  • Lucidchart
  • DbDiagram.io

Ejercicio Guiado

  1. Identifica entidades en un sistema de biblioteca.
  2. Define atributos para cada entidad.
  3. Establece relaciones y cardinalidades.
  4. Dibuja un diagrama E-R básico.
