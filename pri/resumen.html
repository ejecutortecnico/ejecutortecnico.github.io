Bootcamp de Programación Nivel Intermedio

Resumen de Temas de Programación

Esta presentación cubre los temas técnicos de las tres misiones del bootcamp, excluyendo las lecciones de inglés y habilidades de poder.

Usa los botones de navegación para moverte entre las diapositivas.

Misión 1: Desarrollo de Frontend con React

Lección 1: Componentes Funcionales y de Clase

  • Componentización: Dividir la UI en piezas reutilizables e independientes.
  • Componentes Funcionales: Funciones de JavaScript que devuelven JSX. Son el estándar moderno con el uso de Hooks.
  • Componentes de Clase: Clases de ES6 que extienden React.Component. Manejan estado y ciclo de vida con métodos como render() y componentDidMount().
  • Estado (State) y Propiedades (Props): El estado gestiona datos internos que cambian, mientras que las props pasan datos de componentes padres a hijos.

Misión 1: Desarrollo de Frontend con React

Lección 2: Uso de Hooks en Componentes Funcionales

  • useState: Permite a los componentes funcionales manejar su propio estado local. Devuelve el estado actual y una función para actualizarlo.
  • useEffect: Ejecuta efectos secundarios después del renderizado del componente, como llamadas a APIs o suscripciones.
  • Otros Hooks: Se introducen useContext (para estado global sin pasar props), useReducer (para lógica de estado compleja), y useRef (para referencias a elementos del DOM).

Misión 1: Desarrollo de Frontend con React

Lección 3: Manejo de Eventos

  • Eventos Sintéticos: React envuelve los eventos nativos del navegador en un objeto de evento sintético para garantizar la compatibilidad entre navegadores.
  • Manejadores de Eventos: Se usan atributos en camelCase como onClick y onChange para vincular funciones a eventos del DOM.
  • Paso de Parámetros: Se utilizan funciones de flecha para pasar argumentos adicionales a los manejadores de eventos.
  • event.preventDefault(): Evita el comportamiento predeterminado de un evento, como el envío de un formulario.

Misión 1: Desarrollo de Frontend con React

Lección 4: Integración de Conceptos y Aplicación Web Dinámica

  • Planificación de la Aplicación: Diseño de la estructura de componentes, planificación del flujo de datos (unidireccional) y decisión sobre el manejo del estado (local vs. global).
  • Integración de conceptos: Combinar componentes, estado, eventos y hooks en un proyecto funcional.
  • Enrutamiento: Uso de librerías como React Router para crear aplicaciones de una sola página (SPA) con múltiples vistas.

Misión 1: Desarrollo de Frontend con React

Lección 5: Configuración de Proyecto y Componente Principal

  • Creación del Proyecto: Uso de create-react-app para generar la estructura inicial de un proyecto React.
  • Instalación de Dependencias: Manejo de paquetes con npm o yarn (e.g., axios, react-router-dom).
  • Componente App: Estructuración del componente principal que contendrá el resto de la aplicación, incluyendo el estado global y la configuración del enrutador.

Misión 1: Desarrollo de Frontend con React

Lección 6: Desarrollo de Componentes TaskForm y TaskList

  • Componente Controlado: Creación de formularios donde el valor del input está controlado por el estado de React.
  • Comunicación entre Componentes: El componente padre (App) pasa funciones a los componentes hijos (TaskForm, TaskList) a través de props para que estos puedan modificar el estado global.
  • Renderizado de Listas: Uso del método .map() para iterar sobre un array de datos y renderizar un componente por cada elemento. Se requiere una prop key única para cada elemento de la lista.

Lección 7: Integración y Pruebas del Proyecto

  • Pruebas Funcionales: Verificación manual o automatizada de que las funcionalidades clave (agregar, editar, eliminar tareas) operan como se espera.
  • Pruebas Automatizadas: Introducción a herramientas como Jest y React Testing Library para escribir pruebas unitarias y de integración.
  • Refactorización: Proceso de mejorar la estructura y legibilidad del código sin cambiar su funcionalidad externa.
  • Preparación para Despliegue: Optimización de la aplicación para producción, creando una build con npm run build.

Misión 2: Manejo de Bases de Datos con SQL

Lección 1: Introducción a Bases de Datos Relacionales

  • Conceptos Clave: Tabla, Relación, Clave Primaria (Primary Key), Clave Foránea (Foreign Key).
  • DBMS: Un Sistema de Gestión de Bases de Datos (como MySQL, PostgreSQL) es el software para crear, administrar y manipular bases de datos.
  • Creación de BD y Tablas: Uso de los comandos CREATE DATABASE y CREATE TABLE para definir la estructura de la base de datos y sus tablas, especificando columnas y tipos de datos.

Misión 2: Manejo de Bases de Datos con SQL

Lección 2: Manipulación de Bases de Datos

  • Inserción de Datos: INSERT INTO para agregar nuevos registros a una tabla.
  • Actualización de Datos: UPDATE para modificar registros existentes, usando WHERE para especificar qué filas actualizar.
  • Eliminación de Datos: DELETE FROM para eliminar registros. Es crucial usar WHERE para no borrar toda la tabla.
  • Consultas Básicas: SELECT para recuperar datos, FROM para especificar la tabla, y WHERE para filtrar resultados. Uso de ORDER BY para ordenar y LIMIT para restringir el número de resultados.

Misión 2: Manejo de Bases de Datos con SQL

Lección 3: Manejo de Índices y Optimización

  • Índices: Estructuras de datos que mejoran la velocidad de las consultas (SELECT). Se crean en columnas usadas frecuentemente en cláusulas WHERE, JOIN y ORDER BY.
  • Creación de Índices: CREATE INDEX nombre_indice ON nombre_tabla (columna);
  • Ventajas: Aceleran las lecturas.
  • Desventajas: Ralentizan las escrituras (INSERT, UPDATE, DELETE) y ocupan espacio adicional.

Misión 2: Manejo de Bases de Datos con SQL

Lección 4: Consultas Avanzadas

  • Uniones de Tablas (JOIN): Combinan filas de dos o más tablas. Tipos principales: INNER JOIN, LEFT JOIN, RIGHT JOIN.
  • Agregación de Datos: Funciones como COUNT(), SUM(), AVG(), MAX(), MIN(). Se usan con GROUP BY para agrupar resultados y HAVING para filtrar grupos.
  • Subconsultas: Consultas anidadas dentro de otra consulta, a menudo en la cláusula WHERE con IN.

Misión 2: Manejo de Bases de Datos con SQL

Lección 5-7: Simulación - Implementación y Optimización

  • Implementación de Esquema: Crear las tablas para un caso práctico (ej. una tienda online con Clientes, Productos, Pedidos) y establecer relaciones con claves foráneas.
  • Ejecución de Consultas Complejas: Realizar consultas que combinan JOIN, filtrado con WHERE y agregación con GROUP BY para obtener información útil del negocio.
  • Optimización de Consultas: Usar EXPLAIN para analizar el plan de ejecución de una consulta e identificar cuellos de botella. Crear índices estratégicos para acelerar las consultas lentas.

Misión 2: Manejo de Bases de Datos con SQL

Lección 9-11: Proyecto Colaborativo (Co-creación)

  • Planificación: Selección de un sector y diseño del esquema de la base de datos (tablas, columnas, relaciones).
  • Implementación: Creación de la base de datos y las tablas con SQL. Inserción de datos de ejemplo.
  • Desarrollo de Funcionalidades: Escribir scripts SQL para realizar las operaciones principales del sistema (ej. gestión de pacientes, citas, historial médico).
  • Consultas y Reportes: Crear consultas complejas para extraer datos y generar reportes útiles para el análisis.

Misión 3: Desarrollo de Backend con Python y Flask

Lección 1: Introducción a Flask

  • Microframework: Flask es ligero y flexible, sin imponer una estructura estricta.
  • Instalación: Se recomienda usar un entorno virtual (venv) y luego instalar con pip install Flask.
  • Aplicación Básica: Se crea una instancia de Flask(__name__) y se definen rutas con el decorador @app.route('/').
  • Rutas y Vistas: Las rutas son URLs y las vistas son las funciones de Python que se ejecutan para devolver una respuesta (ej. texto, JSON, o HTML).
  • Rutas Dinámicas: Capturan parámetros de la URL, ej. @app.route('/usuario/<nombre>').

Misión 3: Desarrollo de Backend con Python y Flask

Lección 1 (cont.): Vistas y Plantillas

  • Plantillas Jinja2: Flask usa Jinja2 para renderizar HTML dinámico. Permite insertar variables {{ variable }} y usar estructuras de control {% for item in items %}.
  • render_template(): Función de Flask para renderizar un archivo HTML desde la carpeta templates y pasarle variables.
  • Estructura de Proyecto: Se organiza el código en carpetas como /templates para HTML y /static para CSS y JS.

Misión 3: Desarrollo de Backend con Python y Flask

Lección 2: Formularios y Validaciones

  • Manejo de Formularios: Se accede a los datos de un formulario enviado vía POST a través del objeto request.form.
  • Flask-WTF: Extensión que simplifica la creación, validación y renderizado de formularios.
  • Definición de Formularios: Se crean clases que heredan de FlaskForm y se definen los campos (StringField, PasswordField).
  • Validadores: Se usan validadores como DataRequired(), Email(), Length() para asegurar la integridad de los datos.

Misión 3: Desarrollo de Backend con Python y Flask

Lección 2 (cont.): Integración con Bases de Datos

  • SQLAlchemy: Es un ORM (Object-Relational Mapper) que permite interactuar con bases de datos usando objetos de Python en lugar de SQL puro.
  • Flask-SQLAlchemy: Extensión para integrar SQLAlchemy con Flask.
  • Modelos: Se definen clases que heredan de db.Model para representar las tablas de la base de datos. Cada atributo es una columna (db.Column).
  • Operaciones (CRUD): db.session.add(objeto) para crear, Modelo.query.all() para leer, modificar el objeto para actualizar, y db.session.delete(objeto) para borrar. Se confirman los cambios con db.session.commit().

Misión 3: Desarrollo de Backend con Python y Flask

Lección 3: Autenticación y Autorización

  • Autenticación: Proceso de verificar la identidad de un usuario (quién eres).
  • Flask-Login: Extensión para manejar sesiones de usuario (login, logout).
  • Modelo de Usuario: El modelo Usuario debe heredar de UserMixin para ser compatible con Flask-Login.
  • Protección de Rutas: Se usa el decorador @login_required para restringir el acceso a vistas solo para usuarios autenticados.
  • Gestión de Sesión: Las funciones login_user(usuario) y logout_user() manejan el estado de la sesión.

Misión 3: Desarrollo de Backend con Python y Flask

Lección 4-7: Proyecto Integrador y API RESTful

  • Proyecto Aplicación de Blog: Un proyecto completo que integra autenticación de usuarios, gestión de publicaciones (CRUD) y uso de una base de datos.
  • API RESTful: Una interfaz para que diferentes sistemas se comuniquen usando HTTP. Se basa en recursos (ej. `/api/usuarios`) y métodos HTTP (GET, POST, PUT, DELETE).
  • Flask-RESTful: Extensión para crear APIs de manera rápida, definiendo Recursos (clases) que manejan las solicitudes HTTP.
  • Consumo de API: La aplicación web (frontend) consume los datos de la API RESTful (backend) para mostrar y manipular la información.

Misión 3: Desarrollo de Backend con Python y Flask

Lección 9-11: Proyecto Colaborativo (Co-creación)

  • Definición del Proyecto: Selección de un proyecto (ej. Sistema de Gestión de Reservas) y definición de su alcance, requisitos y arquitectura (frontend, backend, BD).
  • Desarrollo Backend (API): Crear los modelos, recursos de la API y rutas necesarias para manejar los datos del proyecto (ej. usuarios, mesas, reservas).
  • Desarrollo Frontend: Crear la interfaz de usuario (con React, Vue, o HTML/CSS/JS simple) que consume la API del backend para mostrar datos e interactuar con el sistema.
  • Integración y Pruebas: Conectar el frontend con el backend y realizar pruebas unitarias, de integración y de aceptación para asegurar el correcto funcionamiento.
