Procedimientos Almacenados y Funciones en SQL

Objetivos

  • Comprender qué son los procedimientos almacenados y las funciones en SQL.
  • Aprender a crearlos y ejecutarlos.
  • Usarlos para automatizar tareas y mejorar la reutilización del código.

¿Qué son Procedimientos y Funciones?

  • Procedimiento almacenado: bloque de código SQL que puede ejecutarse en cualquier momento.
  • Función: devuelve un valor a partir de parámetros de entrada.

Crear un Procedimiento

CREATE PROCEDURE ListarUsuarios()
BEGIN
  SELECT * FROM usuarios;
END;

Ejecutar un Procedimiento

CALL ListarUsuarios();

Crear una Función

CREATE FUNCTION ObtenerEdad(fecha_nac DATE) 
RETURNS INT
BEGIN
  RETURN TIMESTAMPDIFF(YEAR, fecha_nac, CURDATE());
END;

Usar una Función

SELECT nombre, ObtenerEdad(fecha_nac) AS edad
FROM usuarios;

Ventajas

  • Centralizan la lógica de negocio en la base de datos.
  • Mejoran el rendimiento al evitar enviar múltiples consultas.
  • Reutilización del código en varias aplicaciones.

Limitaciones

  • Dependencia del motor de base de datos (MySQL, PostgreSQL, etc.).
  • Pueden ser más difíciles de mantener que el código en la aplicación.

Buenas Prácticas

  • Usar nombres descriptivos en procedimientos y funciones.
  • Documentar parámetros y valores de retorno.
  • Evitar lógica excesivamente compleja dentro de la base de datos.

Ejercicio Guiado

  1. Crea un procedimiento almacenado que devuelva los productos con precio mayor a 100.
  2. Crea una función que calcule el total de una factura dada su ID.
  3. Prueba ambos en tu base de datos.
