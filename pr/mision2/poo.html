¿Qué es la Programación Orientada a Objetos?

Definición: Paradigma de programación basado en el concepto de "objetos" que pueden contener datos (atributos) y código (métodos).

  • Clases
  • Objetos
  • Herencia
  • Encapsulación
  • Polimorfismo

Objetos en JavaScript

En JavaScript, los objetos son colecciones de pares clave-valor:

const persona = {
  nombre: "Juan",
  edad: 30,
  hablar: function() {
    console.log("Hola, soy Juan");
  }
};

Los objetos pueden tener métodos (funciones dentro de ellos).

Clases en JavaScript (ES6)

Antes de ES6, la POO en JavaScript se hacía mediante funciones constructoras. Con ES6, se introdujeron las clases.

class Persona {
  constructor(nombre, edad) {
    this.nombre = nombre;
    this.edad = edad;
  }
  hablar() {
    console.log(`Hola, soy ${this.nombre}`);
  }
}

Crear Objetos usando Clases

const juan = new Persona("Juan", 30);
juan.hablar(); // Hola, soy Juan

new crea una nueva instancia de la clase.

Herencia en JavaScript

class Estudiante extends Persona {
  constructor(nombre, edad, carrera) {
    super(nombre, edad);
    this.carrera = carrera;
  }
  estudiar() {
    console.log(`${this.nombre} está estudiando ${this.carrera}`);
  }
}

extends indica que una clase hereda de otra.

Encapsulación

La capacidad de restringir el acceso a ciertos componentes de un objeto.

class Persona {
  constructor(nombre, edad) {
    let _nombre = nombre;
    this.edad = edad;
    this.getNombre = function() {
      return _nombre;
    }
  }
}

Polimorfismo

Permite que los objetos puedan comportarse de diferentes maneras según su clase.

class Animal {
  hablar() {
    console.log("El animal hace un sonido");
  }
}
class Perro extends Animal {
  hablar() {
    console.log("El perro ladra");
  }
}
const perro = new Perro();
perro.hablar(); // El perro ladra

Beneficios de la POO

  • Reutilización de código (herencia)
  • Modularidad (clases y objetos)
  • Mantenimiento y escalabilidad (encapsulación)
  • Flexibilidad (polimorfismo)

Conclusión

JavaScript es un lenguaje flexible que permite trabajar tanto con un estilo de programación funcional como orientado a objetos.

La Programación Orientada a Objetos permite escribir código más modular y reutilizable.
