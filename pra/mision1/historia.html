Introducción a Node.js y Express

En esta lección conocerás el origen de Node.js, por qué fue creado y por qué es tan importante en el desarrollo web moderno.

A. Introducción a Node.js

Node.js es un entorno de ejecución que permite usar JavaScript fuera del navegador, especialmente para crear servidores y APIs.

Historia de Node.js

Node.js fue creado en 2009 por Ryan Dahl con el objetivo de resolver un problema común: los servidores web tradicionales eran lentos y poco eficientes cuando manejaban muchas conexiones.

Antes de Node.js, servidores como Apache creaban un hilo por cada petición, lo que consumía demasiados recursos cuando miles de usuarios accedían al mismo tiempo.

La idea que inspiró a Ryan Dahl

Dahl notó que aplicaciones como Flickr no podían actualizar en tiempo real una barra de progreso de carga de archivos debido a la forma bloqueante en que funcionaban los servidores.

Esto lo llevó a crear un sistema donde el servidor pudiera atender muchas solicitudes sin quedarse "esperando" a que termine una tarea.

¿Por qué usar JavaScript para Node.js?

  • JavaScript es de un solo hilo, ideal para trabajar con eventos.
  • Ya estaba diseñado para trabajar de forma no bloqueante.
  • Se integró con el motor V8 de Google, extremadamente rápido.

Esto permitió crear servidores rápidos, modernos y muy eficientes.

¿Qué logró Node.js?

Node.js permitió crear aplicaciones capaces de manejar miles de usuarios al mismo tiempo sin bloquear el sistema.

Esto lo hizo ideal para:

  • Chats en tiempo real
  • Aplicaciones colaborativas
  • Juegos online
  • APIs profesionales

Características principales de Node.js

  • Basado en el motor V8 de Google (alto rendimiento).
  • Modelo de ejecución no bloqueante.
  • Arquitectura basada en eventos.
  • Excelente para aplicaciones modernas y escalables.

Modelo No Bloqueante

Node.js puede iniciar una tarea (por ejemplo, leer un archivo) y seguir trabajando en otras solicitudes mientras esa tarea termina.

Esto mejora mucho el rendimiento en sistemas con muchos usuarios conectados.

Event Loop (Bucle de eventos)

Aunque Node.js usa un solo hilo principal, utiliza un sistema llamado Event Loop para atender muchas solicitudes al mismo tiempo.

Esto lo hace eficiente, rápido y capaz de manejar aplicaciones de alto tráfico.

NPM: El ecosistema de paquetes

Node.js incluye NPM (Node Package Manager), una plataforma con millones de librerías listas para usar.

  • Autenticación
  • Seguridad
  • Conexión a bases de datos
  • Frameworks como Express

Node.js es Multiplataforma

Funciona en:

  • Windows
  • Linux
  • macOS

Una misma aplicación puede ejecutarse en cualquier sistema sin modificar el código.

Node.js y desarrollo Full Stack

Con Node.js puedes usar JavaScript tanto en frontend como en backend.

Esto permite a los desarrolladores trabajar con un solo lenguaje en toda la aplicación.

Escalabilidad de Node.js

  • Puede crecer agregando más potencia al servidor (escalado vertical).
  • Puede crecer agregando más servidores (escalado horizontal).
  • Utiliza módulos como Cluster para aprovechar múltiples núcleos del CPU.

Casos de uso reales de Node.js

  • Netflix y YouTube (streaming)
  • PayPal y Walmart (APIs y microservicios)
  • Slack y Trello (colaboración en tiempo real)
  • IoT (Internet de las cosas)
  • Redes sociales y chats

Conclusión sobre Node.js

Node.js transformó el desarrollo web moderno, permitiendo crear aplicaciones:

  • Rápidas
  • Escalables
  • Modernas
  • Profesionales
